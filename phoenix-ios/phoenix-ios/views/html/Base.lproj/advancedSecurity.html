Standard security:

  • Access to the user interface is protected by biometrics.
  • Phoenix can receive incoming payments as long as the app is running.

Advanced security:

  • Access to your seed is protected by biometrics.
  • Phoenix can receive incoming payments as long as the app is running AND you've unlocked the app at least once.

Reduced app functionality:

With advanced security enabled: If the device is restarted, and Phoenix is re-launched automatically (e.g. operating system upgrade), then incoming payments will fail until you unlock Phoenix.

Technical details:

Your seed is wrapped using ChaCha20-Poly1305 with a 256-bit lockingKey, and the resulting {ciphertext, nonce, tag} is stored in a JSON file on disk. The lockingKey is stored in the iOS keychain.

With normal security, the app can access the lockingKey on app launch, and immediately unlock the bitcoin wallet. Enabled security options restrict access to the user interface.

With advanced security, the lockingKey is encumbered such that it cannot be accessed unless you authenticate with biometrics. Thus the app is incapable of unlocking the bitcoin wallet without your assistance.
